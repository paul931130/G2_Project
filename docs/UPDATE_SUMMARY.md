# v1.5.0 整合更新與發布檢查摘要

## 一、版本定位

本版是 v1.5.0的整理發布版，重點不是新增大型功能，而是修正詩庫資料輸出、前端顯示邏輯與發布文件一致性。

## 二、本版修正重點

### 1. 原詩正文與校注分離

原詩 `content` 只保留詩句本體。像 `(唯 通：惟)`、`(明年 一作：年年)`、`(顰 一作：蹙)` 這類校勘資訊，不應混入直排原詩區，而應顯示在「校注／異文」區。

### 2. 詩魂自述輸出清理

清理 `translations` 內的原始標籤，避免以下文字直接顯示給使用者：

- `譯文及注釋`
- `譯文`
- `直譯` / `韻譯` / `意譯` / `漢譯`
- `注釋` / `註解`
- `英譯` / 英文翻譯段落

### 3. 魂歸何處輸出清理

`others` 欄位若只有英譯或不適合作為背景的資料，不再直接顯示為「魂歸何處」。系統會優先取中文創作背景；若背景不足，才回退到中文賞析。

### 4. 長篇題序分離

以下作品的長篇題序已從 `content` 移至 `others`，避免題序出現在原詩直排區：

- 望月有感
- 賊退示官吏
- 石魚湖上醉歌
- 琵琶行 / 琵琶引
- 觀公孫大娘弟子舞劍器行
- 燕歌行

### 5. 快取與版本參數

`poems.json` 讀取時加入版本參數與時間戳，`index.html` 也對 `app.js` 與 `style.css` 加上版本參數，降低 GitHub Pages 或瀏覽器讀到舊版檔案的機率。
