# 詩境｜期末 Demo 影片腳本與現場展示流程

> 建議片長：5～6 分鐘  
> 展示目標：說清楚我做了什麼、為什麼做、學到什麼、哪裡卡住，以及版本如何改善。  
> 影片形式：螢幕錄影 + 口述旁白。  

## 一、影片架構總覽

| 時間 | 畫面 | 重點 |
|---|---|---|
| 0:00–0:30 | 首頁 | 專案名稱與動機 |
| 0:30–1:20 | 搜尋詩作 | 本機詩庫查詢與詩魂呈現 |
| 1:20–2:10 | 黃鶴樓案例 | 校注分離與資料清理 |
| 2:10–3:00 | 隨機、朗讀、收藏 | 不需 API 的展示功能 |
| 3:00–4:00 | API Key 與狀態 | Gemini 延伸與安全設計 |
| 4:00–4:50 | 雙魂對話 | 生成式 AI 互動特色 |
| 4:50–5:50 | 文件與版本紀錄 | 更版歷程、卡住處與反思 |

## 二、逐段旁白腳本

### 0:00–0:30 開場

畫面：打開網站首頁。

旁白：

> 大家好，這是我的生成式 AI 期末專題「詩境」。它是一個古典詩詞互動系統，使用者可以輸入詩名、作者、詩句片段，甚至只輸入一種意境，系統就會召喚對應的詩作。我的目標不是做一個普通查詩網站，而是讓詩以「詩魂現身」的方式被重新閱讀。

### 0:30–1:20 本機詩庫查詢

畫面：輸入 `靜夜思` 或 `黃鶴樓送孟浩然之廣陵`，按下查詢。

旁白：

> 這個系統有兩種資料來源。第一種是本機詩庫，也就是 `poems.json`，目前整理了 315 首詩。即使沒有輸入 API Key，也可以直接查詢本機詩庫，所以展示時不會完全依賴 API 額度。查到作品後，右側會出現詩魂現身，左側會顯示原詩，並提供白話釋義、情緒分析與背景說明。

### 1:20–2:10 校注分離案例

畫面：展示《黃鶴樓送孟浩然之廣陵》。

旁白：

> 這次改版中，我花最多時間處理資料輸出問題。原本詩庫資料裡，有些原詩會混入校勘註記，例如「唯 通：惟」。如果直接顯示，直排詩文就會變得很奇怪，也會讓使用者誤以為那是詩句本身。現在新版會自動把這類內容從原詩中分離，原詩只顯示真正詩句，校注則放在獨立區塊。

可補一句：

> 這讓作品從「能跑」變成比較接近「可發布」的狀態。

### 2:10–3:00 隨機召喚、朗讀、收藏

畫面：按「隨機一首」、按朗讀、收藏詩作。

旁白：

> 除了查詢，我也加入了幾個展示用功能。第一個是隨機召喚，使用者不用輸入內容，就能從本機詩庫抽出一首詩。第二個是朗讀，這裡使用瀏覽器內建的 Web Speech API，所以不用另外申請 ElevenLabs 或語音服務。第三個是收藏，收藏會存在瀏覽器的 localStorage，方便之後重新打開。

### 3:00–4:00 Gemini API 與安全設計

畫面：點 API KEY，展示狀態列。若不方便輸入 key，可只展示位置與說明。

旁白：

> 如果使用者輸入 Gemini API Key，系統可以做更進一步的 AI 分析與詩魂問答。安全上，我沒有把 API Key 寫進程式，也不會上傳到 GitHub，而是只存在目前分頁的 sessionStorage，關掉分頁後就會清除。系統也有 API 狀態列，如果遇到配額限制或 429，會進入冷卻，不會一直重複測試。

### 4:00–4:50 雙魂對話

畫面：切到雙魂對話，選李白與杜甫，主題輸入「離別」。

旁白：

> 這個功能是雙魂對話。使用者可以選兩位詩人，再輸入一個主題，讓 AI 生成兩位詩人之間的對話。這是專案比較有生成式 AI 特色的地方，因為它不只是輸出資料，而是用角色設定讓古典文本產生互動。

### 4:50–5:50 更版紀錄與反思

畫面：打開 `CHANGELOG.md` 或 GitHub Release 頁面。

旁白：

> 這個專案從初版一路改到 v1.5.0。前面先完成基本 AI 查詩，接著拆分檔案、加入本機詩庫、隨機召喚、朗讀、收藏與雙魂對話。後面主要是在修穩定性，包括 API 配額、手機版排版、快取、校注分離、譯文標籤清理與長序文分離。最大的收穫是，我發現 AI 專案不是接上 API 就結束，真正困難的是資料清理、錯誤處理、部署快取與使用者體驗。這也是這次專案最重要的學習。

結尾：

> 最後，詩境的特色是把 AI、本機資料庫與詩詞閱讀氛圍整合在一起，讓古典詩不只是被查詢，而是可以被召喚、被追問、被聽見。

## 三、現場展示備案

如果現場 API 失效或 Gemini 配額受限，展示順序改成：

1. 本機查詢 `黃鶴樓送孟浩然之廣陵`
2. 隨機召喚
3. 朗讀
4. 收藏與複製
5. 打開 `CHANGELOG.md` 說明 Gemini API 可用時會啟用詩魂問答與雙魂對話

這樣即使 API 失效，也仍然可以完整展示專案核心。

## 四、錄影前檢查清單

- [ ] 使用最新版 v1.5.0 發布包
- [ ] GitHub Pages 已更新完成
- [ ] 手機或瀏覽器不要停留在舊快取頁面
- [ ] 測試 `黃鶴樓送孟浩然之廣陵` 不出現 `(唯 通：惟)`
- [ ] 測試 `怨情` 不出現 `譯文及注釋` 或 `英譯`
- [ ] 測試 `琵琶行` 題序不在原詩直排區
- [ ] 測試隨機召喚、朗讀、收藏
- [ ] 若要展示 Gemini，先確認 API Key 可用
